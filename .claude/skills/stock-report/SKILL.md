# stock-report — รายงานสถานะสต็อก StockApp

Skill นี้ใช้สร้าง **รายงานสถานะสต็อกในรูปแบบมาตรฐาน** เมื่อผู้ใช้ถามเรื่องสรุปสต็อก สินค้าใกล้หมด รายงานคลังสินค้า หรือ stock summary

## แหล่งข้อมูล (Data source)

ดึงข้อมูลสินค้าจากฐานข้อมูลเสมอ — **ต้อง filter `deletedAt: null`** (ไม่นับสินค้าที่ soft-delete แล้ว)

- ผ่าน MCP: `mcp__postgres__query` (read-only) — query ตาราง `"Product"`
- หรือผ่าน helper ในโค้ด: `@lib/queries.ts`

ตัวอย่าง SQL (read-only) — สังเกตว่า **ไม่มี semicolon ในตัวอย่างโค้ด**:

```sql
SELECT sku, name, quantity, "reorderPoint"
FROM "Product"
WHERE "deletedAt" IS NULL
ORDER BY quantity ASC
```

## กฎการจัดสถานะ (Status rules)

จัดสถานะแต่ละรายการตามลำดับความสำคัญนี้ (บนสุดชนะก่อน):

1. `quantity == 0` → **🔴 หมดแล้ว**
2. `quantity <= reorderPoint` (แต่ยังมากกว่า 0) → **⚠️ ใกล้หมด**
3. นอกเหนือจากนั้น → **✅ ปกติ**

หมายเหตุ: เงื่อนไข "หมดแล้ว" ครอบไว้ก่อน "ใกล้หมด" เพราะ `quantity == 0` เข้าเงื่อนไข `<= reorderPoint` ด้วย

ตัวอย่าง logic (no semicolon):

```ts
function getReportStatus(quantity: number, reorderPoint: number) {
  if (quantity === 0) return "🔴 หมดแล้ว"
  if (quantity <= reorderPoint) return "⚠️ ใกล้หมด"
  return "✅ ปกติ"
}
```

## รูปแบบรายงาน (Report format)

ออกรายงานเป็น Markdown ตามโครงนี้เป๊ะ ๆ:

### 1. หัวข้อ + วันที่

```markdown
# รายงานสถานะสต็อก
**วันที่:** 12 กรกฎาคม 2026
```

ใช้วันที่ปัจจุบัน (โซนเวลาไทย) รูปแบบ `วัน เดือน(ไทย) ปี พ.ศ. หรือ ค.ศ.` — ให้สอดคล้องกับ `@lib/format.ts`

### 2. ตารางรายการสินค้า

เรียงลำดับให้รายการที่ต้องสนใจอยู่บนสุด: **🔴 หมดแล้ว → ⚠️ ใกล้หมด → ✅ ปกติ**

| SKU | ชื่อสินค้า | คงเหลือ | จุดสั่งซื้อ | สถานะ |
|-----|-----------|--------:|-----------:|-------|
| `SKU-001` | ตัวอย่างสินค้า A | 0 | 10 | 🔴 หมดแล้ว |
| `SKU-002` | ตัวอย่างสินค้า B | 5 | 10 | ⚠️ ใกล้หมด |
| `SKU-003` | ตัวอย่างสินค้า C | 120 | 20 | ✅ ปกติ |

- คอลัมน์ **คงเหลือ** = `quantity`, **จุดสั่งซื้อ** = `reorderPoint`
- จัดคอลัมน์ตัวเลขชิดขวา
- SKU แสดงเป็น mono (`` `...` ``)

### 3. สรุปท้ายรายงาน (Summary)

ปิดท้ายด้วยบล็อกสรุปเสมอ:

```markdown
## สรุป
- **จำนวนสินค้าทั้งหมด:** 3 รายการ
- **🔴 หมดแล้ว:** 1 รายการ
- **⚠️ ใกล้หมด:** 1 รายการ
- **รายการที่ต้องสั่งซื้อ (หมด + ใกล้หมด):** 2 รายการ
  - `SKU-001` ตัวอย่างสินค้า A (คงเหลือ 0 / จุดสั่งซื้อ 10)
  - `SKU-002` ตัวอย่างสินค้า B (คงเหลือ 5 / จุดสั่งซื้อ 10)
```

- **จำนวนสินค้าทั้งหมด** = จำนวน row ที่ `deletedAt IS NULL`
- **รายการที่ต้องสั่งซื้อ** = ทุกรายการที่สถานะเป็น 🔴 หรือ ⚠️ (คือ `quantity <= reorderPoint`) — ไล่รายชื่อพร้อมคงเหลือ/จุดสั่งซื้อกำกับ
- ถ้าไม่มีรายการที่ต้องสั่งซื้อเลย ให้เขียนว่า **"✅ สต็อกทุกรายการอยู่ในระดับปกติ ไม่มีสินค้าที่ต้องสั่งซื้อ"**

## กฎเด็ดขาด

- ⛔ **ไม่มี semicolon ในตัวอย่างโค้ดใด ๆ** (TypeScript/JavaScript) — กฎของโปรเจกต์นี้
- ทุกข้อความในรายงานเป็น **ภาษาไทย**
- ห้าม hardcode ตัวเลข — ดึงจากฐานข้อมูลจริงเสมอ และ filter `deletedAt: null` ทุกครั้ง
